## Project Overview

PortfolioIQ is a local Investment Portfolio Monitor & Recommendation Engine.
It connects to Interactive Brokers (IBKR) via TWS API, syncs portfolio data
automatically, and provides AI-powered buy/sell recommendations through a
modern fintech-style web UI.

All data stays local — no cloud backend, no external storage.

## Tech Stack

| Layer | Technology |
|---|---|
| Backend | Python 3.12+, FastAPI, SQLAlchemy (async), Alembic, APScheduler |

| Package Mgr | uv (Python), npm (frontend) |

## Conventions

### Python
- **All async**: use `async def` for FastAPI routes and service methods
- **Models**: SQLAlchemy mapped classes in `backend/models/`; one file per table
- **Schemas**: Pydantic request/response models co-located in router files, or in a `schemas/` module if shared
- **Config**: all env-driven settings in `backend/config.py` via `pydantic-settings`; never import `os.environ` directly elsewhere
- **Format**: `ruff format backend/` — line length 100
- **Lint**: `ruff check backend/`
- **Type check**: `mypy backend/`
- **Tests**: `uv run pytest` — all tests in `tests/`, use `pytest-asyncio` for async tests

### Frontend
- All components in TypeScript `.tsx`
- Use **TanStack Query** (`useQuery`, `useMutation`) for all API calls — never `useEffect` + `fetch`
- **shadcn/ui** components live in `src/components/ui/` (generated by `npx shadcn@latest add`)
- Custom components in `src/components/{feature}/`
- **Dark theme** by default; CSS variables in `src/index.css`
- Path alias `@/` maps to `src/`
- `clsx` + `tailwind-merge` for conditional class names via `cn()` utility

### Commits
- Conventional commits: `feat:`, `fix:`, `docs:`, `chore:`, `refactor:`
- Keep commits focused; one logical change per commit

## Running the Dev Environment

### Prerequisites
1. **TWS or IB Gateway** running locally on port `4002` (paper) or `4001` (live)
   - TWS: Settings → API → Enable ActiveX and Socket Clients → allow localhost
2. Python 3.12+ and `uv` installed
3. Node 20+ and `npm` installed
4. Copy `.env.example` to `.env` and fill in your IBKR account number and Anthropic API key

### Backend
```bash
uv sync
uv run uvicorn backend.main:app --reload --port 8000
```

### Frontend
```bash
cd frontend
npm install
npm run dev
```

The Vite dev server (port 5173) proxies all `/api` requests to `localhost:8000`.

## Key Architectural Notes

- **SQLite file** (`portfolioiq.db`) is gitignored — never commit it
- **IBKR sync** runs automatically every 5 minutes via APScheduler once the backend starts; call `POST /api/portfolio/sync` to trigger manually
- **Quote cache**: 60s TTL during market hours, 15 min after hours; served from SQLite `market_data` table
- **Recommendation Engine**: powered by Claude (`claude-sonnet-4-5`) — synthesizes raw strategy signals + portfolio context + user preferences → generates `SmartRecommendation` with action (BUY/ADD/TRIM/SELL/HOLD), sizing, and rationale
- **Strategy hierarchy**: raw signals (momentum, value, growth, etc.) → hybrid profiles (user-defined weighted blends) → AI recommendation engine
- ALWAYS refer to `docs/ARCHITECTURE.md` for full system design and DB schema
- ALWAYS refer to  `docs/STRATEGIES.md` for strategy logic and confidence scoring formulas
- ALWAYS refer to  `docs/TASKS.md` for the Iteration 1 task breakdown
